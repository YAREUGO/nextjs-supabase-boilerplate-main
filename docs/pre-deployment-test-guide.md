# 배포 전 테스트 가이드

이 문서는 배포 전에 수동으로 테스트해야 할 주요 기능들을 정리한 가이드입니다.

## 🧪 테스트 체크리스트

### 1. 회원가입/로그인 플로우
- [ ] 회원가입이 정상적으로 작동하는지 확인
- [ ] 로그인이 정상적으로 작동하는지 확인
- [ ] 로그아웃이 정상적으로 작동하는지 확인
- [ ] Clerk 모달이 정상적으로 표시되는지 확인

### 2. 상품 조회 및 필터링
- [ ] 홈페이지에서 상품이 정상적으로 표시되는지 확인
- [ ] 상품 목록 페이지가 정상적으로 로드되는지 확인
- [ ] 카테고리 필터가 정상적으로 작동하는지 확인
- [ ] 상품 상세 페이지가 정상적으로 표시되는지 확인
- [ ] 상품 이미지가 정상적으로 표시되는지 확인

### 3. 장바구니 기능
- [ ] 상품을 장바구니에 추가할 수 있는지 확인
- [ ] 장바구니 페이지에서 아이템이 정상적으로 표시되는지 확인
- [ ] 장바구니 아이템 수량 변경이 정상적으로 작동하는지 확인
- [ ] 장바구니 아이템 삭제가 정상적으로 작동하는지 확인
- [ ] 장바구니 총 금액이 정상적으로 계산되는지 확인
- [ ] 로그인하지 않은 사용자가 장바구니에 추가 시도 시 로그인 안내가 표시되는지 확인

### 4. 주문 프로세스
- [ ] 주문 페이지가 정상적으로 로드되는지 확인
- [ ] 배송지 정보 입력 폼이 정상적으로 작동하는지 확인
- [ ] 주문 요약이 정상적으로 표시되는지 확인
- [ ] 주문 유효성 검증이 정상적으로 작동하는지 확인
- [ ] 주문 생성이 정상적으로 완료되는지 확인

### 5. 결제 프로세스
- [ ] 결제 페이지가 정상적으로 로드되는지 확인
- [ ] Toss Payments 위젯이 정상적으로 표시되는지 확인
- [ ] 결제 수단 선택이 정상적으로 작동하는지 확인
- [ ] 테스트 결제가 정상적으로 완료되는지 확인
- [ ] 결제 성공 후 주문 상태가 업데이트되는지 확인
- [ ] 결제 실패 시 적절한 에러 메시지가 표시되는지 확인
- [ ] 결제 성공 후 장바구니가 비워지는지 확인

### 6. 마이페이지 기능
- [ ] 마이페이지가 정상적으로 로드되는지 확인
- [ ] 주문 내역이 정상적으로 표시되는지 확인
- [ ] 주문 상태가 정상적으로 표시되는지 확인
- [ ] 주문 상세 페이지가 정상적으로 표시되는지 확인
- [ ] 주문 상세에서 배송지 정보가 정상적으로 표시되는지 확인
- [ ] 주문 상세에서 결제 정보가 정상적으로 표시되는지 확인

## 🔍 에러 시나리오 테스트

### 네트워크 오류
- [ ] 네트워크 연결이 끊겼을 때 적절한 에러 메시지가 표시되는지 확인
- [ ] API 요청 실패 시 사용자에게 친화적인 메시지가 표시되는지 확인

### 권한 오류
- [ ] 로그인하지 않은 사용자가 보호된 페이지에 접근 시 적절한 안내가 표시되는지 확인
- [ ] 다른 사용자의 주문에 접근 시도 시 적절한 에러 처리가 되는지 확인

### 데이터 오류
- [ ] 존재하지 않는 상품 ID로 접근 시 404 페이지가 표시되는지 확인
- [ ] 존재하지 않는 주문 ID로 접근 시 적절한 에러가 표시되는지 확인
- [ ] 재고가 없는 상품을 장바구니에 추가 시도 시 적절한 메시지가 표시되는지 확인

## 🐛 알려진 이슈

### Realtime 인증 토큰 경고
빌드 시 다음 경고가 나타날 수 있습니다:
```
Failed to set initial Realtime auth token: TypeError: a is not a function
```

**영향**: 없음 (런타임에 영향을 주지 않음)
**원인**: Supabase Realtime 기능이 초기화될 때 발생하는 경고
**조치**: Realtime을 사용하지 않으므로 무시해도 됩니다.

### Dynamic Server Usage 경고
빌드 시 다음 경고가 나타날 수 있습니다:
```
Route / couldn't be rendered statically because it used `cookies`.
```

**영향**: 없음 (정상적인 동작)
**원인**: Clerk 인증을 사용하는 페이지는 동적으로 렌더링되어야 함
**조치**: 정상적인 동작이므로 무시해도 됩니다.
